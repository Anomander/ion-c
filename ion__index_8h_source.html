00015 /*
00016  * this helps define indexed collections used for symbol support
00017  * in Ion.c.  These follow the collection pattern and use the base
00018  * collection heavily.  And, like ion_index, the memory used 
00019  * for the nodes is allocated on the parent, which is passed in when 
00020  * the user initializes an index.
00021  *
00022  * each collections holds a high water mark free list of nodes that
00023  * were previously used but aren't currently being used
00024  *
00025  * index supports:
00026  *    iERR  initialize(ION_INDEX *idx, CMP_FN cmp, HASH_FN hash)
00027  *    BOOL  exists    (void *key)
00028  *    void *find      (void *key)
00029  *    BOOL  insert    (void *key, void *data)
00030  *    BOOL  upsert    (void *key, void *data)
00031  *    void  delete    (void *key)
00032  *    void  reset     ()
00033  *    void  release   ()
00034  *
00035  * unlike collection index expects the caller to own the key and
00036  * the data objects and index itself only maintains the additional
00037  * data to manage these functions.
00038  *
00039  * there are also cursor macros that mirror the collection cursors
00040  *
00041  * to define the index comparison behavior the user supplies a
00042  * compare function and a hash function
00043  */
